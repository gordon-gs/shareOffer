# 共享报盘应用工作模型
```text
            +------------------+
            |      交易网关      |      <---- TCP_SERVER  ( tcp_conn_gw1 ... tcp_conn_gw2 )
            +------------------+
                      |
                      |
              192.168.56.11:5201  | MAC0
                      |
             +------------------+                   <- 应用接口作服务端( tcp_conn_t[3] )
             | 共享报盘应用      |     <---- 在用户线程中等待事件/收发数据
             | 192.168.56.1 MAC0|
             |                  |
             | 192.168.99.1 MAC1|
             +------------------+                   <- 底层sock接口( SOCK/INSTANTA_SOCK/TOE_SOCK )
        :18000  /       |  :18001
               /        |
    +----------+   +----------+
    | 柜台1     |   |  柜台2   |      <---- 应用接口作客户端端 ( tcp_conn_t[1], tcp_conn_t[2], ... )
    +----------+   +----------+
```

功能演示:
1. 程序启动后主动连接行情主机建立 tcp_conn_gw1
2. 程序启动后提供 TCP端口供外部客户端连接, tcp_conn_c1, tcp_conn_c2
3. 当 **交易网关** 上线后, 等待 **交易柜台** 上线, 在**交易网关**与**交易柜台**之间进行双向转发,完成 ping-pong
