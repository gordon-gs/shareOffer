# NDPP-TOE-Socket 项目总结文档

## 项目概述

**项目名称**: NDPP-TOE-Socket  
**当前版本**: 0.2.0  
**开发单位**: YUSUR Technology Co., Ltd.  
**项目类型**: TCP 连接管理库

本项目是一个高性能的 TCP 连接管理库（libtcp_conn），提供了统一的异步 I/O 接口，支持客户端和服务端的 TCP 连接管理。该库基于 epoll 实现了异步非阻塞的数据收发，具有低延迟、高吞吐的特点。

#### 2.3 RX/TX 缓冲区管理

基于环形缓冲区和管道实现异步数据收发：

**RX 缓冲区 (rx_buffer_t)**：

```c
// tcp_buffer_rx_tx.h
typedef struct rx_buffer_s
{
    int             pipe_fd[2];    // 用于事件通知
    ringbuff_t      buffer;        // 环形缓冲区
    pthread_mutex_t lock;          // 线程锁
    bool            ready;         // 就绪标志
} rx_buffer_t;

// RX 线程从 socket 读取数据，写入 rx_buffer
// 通过 pipe_fd[1] 通知应用数据就绪
// 应用通过 epoll 监听 pipe_fd[0]
```

**TX 缓冲区 (tx_buffer_t)**：

```c
typedef struct tx_buffer_s
{
    int             pipe_fd[2];    // 用于事件通知
    ringbuff_t      buffer;        // 环形缓冲区
    pthread_mutex_t lock;
    bool            ready;
} tx_buffer_t;

// 应用调用 tcp_conn_send，数据写入 tx_buffer
// 通过 pipe_fd[1] 通知 TX 线程
// TX 线程从 tx_buffer 读取数据发送到 socket
```

#### 2.4 线程模型

采用专用的 RX/TX 线程处理 I/O：

```c
// rx_thread_func - 接收线程
static void *rx_thread_func(void *arg)
{
    tcp_conn_manage_t *mgr = (tcp_conn_manage_t *)arg;
    struct epoll_event events[MAX_EVENTS];

    while (mgr->is_running)
    {
        // epoll_wait 监听所有连接的 socket
        int n = epoll_wait(mgr->rx_epoll_fd, events, MAX_EVENTS, 100);
        
        for (int i = 0; i < n; i++)
        {
            tcp_conn_item_t *conn = events[i].data.ptr;
            // 从 socket 读取数据
            ssize_t len = recv(conn->io_fd, buffer, sizeof(buffer), 0);
            if (len > 0)
            {
                // 写入 rx_buffer
                rx_buffer_write(&conn->rx_buf, buffer, len);
                // 发送 TCP_EVENT_RX_READY 事件
                notify_event(conn, TCP_EVENT_RX_READY);
            }
        }
    }
}

// tx_thread_func - 发送线程
static void *tx_thread_func(void *arg)
{
    tcp_conn_manage_t *mgr = (tcp_conn_manage_t *)arg;
    struct epoll_event events[MAX_EVENTS];

    while (mgr->is_running)
    {
        // epoll_wait 监听所有连接的 tx_buffer pipe
        int n = epoll_wait(mgr->tx_epoll_fd, events, MAX_EVENTS, 100);
        
        for (int i = 0; i < n; i++)
        {
            tcp_conn_item_t *conn = events[i].data.ptr;
            const void *data;
            size_t len;
            // 从 tx_buffer 获取数据
            if (tx_buffer_peek_next(&conn->tx_buf, &data, &len) == 0)
            {
                // 发送到 socket
                ssize_t sent = send(conn->io_fd, data, len, 0);
                if (sent > 0)
                {
                    tx_buffer_consume(&conn->tx_buf, sent);
                }
            }
        }
    }
}
```

**使用示例（数据收发流程）**：

```c
// 获取事件通知 fd，加入 epoll 监听
int event_fd = tcp_conn_get_event_fd(conn);
struct epoll_event ev;
ev.data.ptr = conn;
ev.events = EPOLLIN;
epoll_ctl(epfd, EPOLL_CTL_ADD, event_fd, &ev);

// epoll_wait 等待事件
struct epoll_event events[32];
int n = epoll_wait(epfd, events, 32, -1);

for (int i = 0; i < n; i++)
{
    tcp_conn_item_t *conn = events[i].data.ptr;
    int event_fd = tcp_conn_get_event_fd(conn);
    
    // 读取事件类型
    tcp_conn_event_t evt;
    read(event_fd, &evt, sizeof(evt));
    
    if (evt.type == TCP_EVENT_RX_READY)
    {
        const void *data;
        int len;
        
        // 零拷贝接收数据
        if (tcp_conn_recv(conn, &data, &len) == 0 && len > 0)
        {
            // 处理数据
            printf("Received %d bytes: %.*s\n", len, len, (char *)data);
            
            // 回显数据
            tcp_conn_send(conn, data, len);
            
            // 标记数据已处理
            tcp_conn_consume(conn, len);
        }
    }
}
```

### 1. TCP Server Echo 示例

**功能**：接收客户端数据并原样回显

```c
// tcp_server_main.c (简化版)
int main(int argc, char *argv[])
{
    // 1. 初始化，加载通道配置
    g_tcp_mgr = tcp_conn_mgr_create("loopback_config.json");

    // 4. 启动事件处理线程
    pthread_create(&event_thread, NULL, tcp_event_thread, NULL);
    
    // 5. 主线程等待
    while (g_running)
    {
        sleep(1);
    }
    
    // 6. 清理资源
    tcp_conn_close(g_server_conn);
    tcp_conn_mgr_destroy(g_tcp_mgr);
    
    return 0;

// 事件处理线程
void *tcp_event_thread(void *arg)
{
    int epfd = epoll_create1(0);
    int event_fd = tcp_conn_get_event_fd(g_server_conn);
    
    // 添加到 epoll
    struct epoll_event ev;
    ev.events = EPOLLIN;
    ev.data.ptr = g_server_conn;
    epoll_ctl(epfd, EPOLL_CTL_ADD, event_fd, &ev);
    
    while (g_running)
    {
        struct epoll_event events[32];
        int nfds = epoll_wait(epfd, events, 32, 1000);
        
        for (int i = 0; i < nfds; i++)
        {
            tcp_conn_item_t *conn = events[i].data.ptr;
            tcp_conn_event_t event;
            read(tcp_conn_get_event_fd(conn), &event, sizeof(event));
            
            if (event.type == TCP_EVENT_RX_READY)
            {
                const void *data;
                int data_len;
                
                // 接收数据
                tcp_conn_recv(conn, &data, &data_len);
                
                // 回显数据
                tcp_conn_send(conn, data, data_len);
                
                // 消费数据
                tcp_conn_consume(conn, data_len);
            }
        }
    }
}
```

### 3. TCP 双向转发应用

**功能**：在两个 TCP 连接之间转发数据

```c
// tcp_share_forward_app_main.c (简化版)
void *tcp_forward_app_thread(void *arg)
{
    tcp_conn_manage_t *mgr = (tcp_conn_manage_t *)arg;
    int epfd = epoll_create1(0);
    
    // 监听两个连接的事件
    for (int i = 0; i < 2; i++)
    {
        int event_fd = tcp_conn_get_event_fd(g_tcp_conn_list[i]);
        struct epoll_event ev;
        ev.data.ptr = g_tcp_conn_list[i];
        ev.events = EPOLLIN;
        epoll_ctl(epfd, EPOLL_CTL_ADD, event_fd, &ev);
    }
    
    while (1)
    {
        struct epoll_event events[32];
        int n = epoll_wait(epfd, events, 32, -1);
        
        for (int i = 0; i < n; i++)
        {
            tcp_conn_item_t *conn = events[i].data.ptr;
            tcp_conn_event_t evt;
            read(tcp_conn_get_event_fd(conn), &evt, sizeof(evt));
            
            if (evt.type == TCP_EVENT_RX_READY)
            {
                const void *data;
                int len;
                
                // 从源连接接收数据
                tcp_conn_recv(conn, &data, &len);
                
                // 转发到目标连接
                int src_id = evt.conn_id;
                int dst_id = (src_id == 0) ? 1 : 0;
                tcp_conn_item_t *dst_conn = tcp_conn_find_by_id(mgr, dst_id);
                
                if (tcp_conn_state(dst_conn) == CONN_STATE_CONNECTED)
                {
                    tcp_conn_send(dst_conn, data, len);
                }
                
                tcp_conn_consume(conn, len);
            }
        }
    }
}

int main(int argc, char *argv[])
{
```

## 性能特性

### 1. 零拷贝机制

- **RX 路径**：Socket → RX Buffer → 应用（通过指针直接访问）
- **TX 路径**：应用 → TX Buffer → Socket
- **优势**：减少内存拷贝次数，降低 CPU 开销

### 2. 异步非阻塞 I/O

- 基于 epoll 实现事件驱动
- 专用的 RX/TX 线程处理 I/O
- 应用线程仅处理业务逻辑

### 3. 环形缓冲区优化

- 缓存行对齐 (64 字节填充)
- volatile 关键字保证多线程可见性
- 无锁读写操作（在单生产者单消费者模型下）

### 4. 管道通知机制

- 使用 pipe 实现跨线程事件通知
- epoll 监听 pipe 读端，实现高效事件分发
- 避免轮询和忙等待

## 版本历史

### v0.2.0 (2025-10-24)

**API 变更**：
- 调整管理器操作接口：`tcp_conn_mgr_create` 需要传入配置文件路径
- 移除 `tcp_conn_mgr_start` 方法
- 调整管理TCP连接接口：`tcp_conn_connect/listen` 只需传入 `tcp_conn` 对象
- 新增获取TCP连接配置和信息接口：`tcp_conn_get_info`
- 新增接口获取版本信息：`tcp_conn_lib_version`
- 新增接口获取错误码文字信息：`tcp_conn_strerror`

**特性变化**：
- 在 `tcp_conn_type.h` 中定义并返回TCP连接状态变化，可从消息通知中获取
- TCP 连接管理器初始化需从配置文件加载期望的连接参数
- libev 编译依赖从 instanta/socket 版本中移除
- 作为TCP Server时，示例配置文件支持为多个通道指定相同本地端口，由运行时自动合并为1路侦听

### v0.1.0 (2025-07-17)

**架构变更**：
- TCP连接管理等网络操作被统一封装，作为独立库 (libtcp_conn)
- 应用只需与简化的API接口交互 (10个关键函数)
- 所有底层细节 (套接字管理、I/O线程、缓冲) 由库内部处理

**优势**：
- 降低应用代码复杂度
- 跨应用行为一致
- 通过相同接口支持C和Rust应用

**软件包**：
- host/tcp_conn_lib：libtcp_conn 静态库与头文件
- host/tcp_share_app：C 示例
- host/tcp_share_app_rust：Rust FFI 示例

## 总结

NDPP-TOE-Socket 项目是一个设计精良、高性能的 TCP 连接管理库。通过统一的 API 接口、事件驱动模型和零拷贝机制，实现了低延迟、高吞吐的网络通信。项目支持多种应用场景，从简单的 Echo 服务到复杂的数据转发系统，都能轻松应对。

**核心价值**：
1. **降低开发复杂度**：应用开发者无需关心底层网络细节
2. **统一接口**：支持 C/Rust 等多种语言
3. **高性能**：epoll + 零拷贝 + 异步 I/O
4. **易于扩展**：插件式后端支持（Socket/Instanta/TOE）

该项目适用于对网络性能有高要求的场景，如金融交易、实时数据分发、高性能代理等。
